The doorway

Paste one line

~30 seconds of your time

You open the Terminal app and paste a single command. macOS asks for your Mac password once — the same prompt you see installing any app — to authorize adding software.

On your screen — the Terminal app
Terminal — bash — 80×24
your-mac:~ you$ curl -fsSL https://raw.githubusercontent.com/craigfilek/fpm-agentic-os/main/install.sh | bash Password: •••••••• ← macOS asks once, then you walk away

You

  • Paste the line, press Enter
  • Type your Mac password once
  • Walk away

The system

  • Downloads install.sh and starts running it
  • Every step checks "already done?" first — safe to re-run
Safety note: nothing here reads your files or phones home with your data. The script installs standard tools and asks for keys with a hidden prompt that writes only to local files (locked to you, chmod 600). No secret is ever typed on the command line or committed anywhere.

Walk away

The system builds itself

~30 minutes · fully automatic · you're not here

Hands-off. The installer lays down the whole engine in order, checking each piece before it acts.

On your screen — the install scrolling by
Terminal — install.sh
== Homebrew == brew present: Homebrew 4.x == core tools (git, gh, ripgrep, ffmpeg) == == Claude Code == == bun == == uv == == gbrain == gbrain: 0.9.x == clone component repos == fpm-ai · hermes-agent · stack-primer · gstack == Anthropic API key == Paste your Anthropic API key (input hidden): •••••••• == doctor (verifying the install end-to-end) == brain OK · sync OK · recall OK READY OK == install.sh complete == Next: open Claude Code and run /onboard

You

  • Nothing. Get a coffee.

The system installs & wires

  • Homebrew (the one password prompt)
  • Runtimes: Bun (JavaScript), uv (Python)
  • Tools: git, gh, ripgrep, ffmpeg, markitdown, yt-dlp
  • Claude Code (your assistant)
  • Clones the component repos: fpm-ai, hermes-agent, stack-primer, gstack
  • gbrain — your local memory (no cloud, runs on your Mac)
  • Seeds an Obsidian vault (Inbox/ + Distilled/)
  • Registers gbrain as a read-only search tool inside Claude Code & Hermes
  • Lays a safety baseline into ~/.claude (auto-compact off, delete/push guards)
  • Runs bin/doctor to prove the install end-to-end

Two keys are collected mid-install, each via a hidden prompt (or skip and add later, see KEYS.md): your Anthropic API key (required — the brain) and a ZeroEntropy key (recommended — upgrades recall from keyword to search-by-meaning).

Meet your CEO — the interview

~15 minutes · this is the only part you actively do

You come back to a terminal with the CEO interview running. It talks like a sharp 11th-grader, asks one question at a time, reflects each answer back in a sentence, and pushes vague answers ("give me a specific example"). Every answer becomes part of your system. Here is the full set, and where each one lands.

It writes you down

Your answers become the system

~1–2 minutes · automatic, with your sign-off

The CEO writes your words (lightly cleaned, never paraphrased) into the four core files plus the long-form context, then seeds your memory. It shows you the titles and asks "Approve, edit, or scrap?" — nothing finalizes until you say so.

On your screen — the sign-off + the proof
Claude Code — writing your system
Writing your core files: ✓ CLAUDE.md # your operating manual ✓ SOUL.md # who you are at the core ✓ FOCUS.md # north star + this month ✓ memory/MEMORY.md # the starting brain Approve, edit, or scrap? › approve $ bin/test-onboard 4 core files present · non-empty · anchors OK READY OK ✓ — you're set up

It builds

  • CLAUDE.md — your operating manual (identity, voice, decision rights, priorities). Every agent reads this first.
  • SOUL.md — who you are at the core: values, voice, what the work is for.
  • FOCUS.md — north star + this month's 2–3 priorities + first build.
  • memory/MEMORY.md + one short memory per durable preference.
  • context/*, connections.md, and a dated line in decisions/log.md.

You

  • Read the titles
  • Say approve, edit, or scrap

Then it proves it: runs bin/test-onboard — green means all four core files exist, are non-empty, and carry their anchors. If gbrain is installed, each context file is indexed into your brain so recall works over your own answers.

Your answers stay yours. Everything the CEO writes (CLAUDE.md, context/*) is git-ignored by the kit, so your personal answers never ship even if you push the repo. They live only on your machine.

It's alive

Day one: two verbs

ongoing · seconds per use

The whole system runs on two verbs. That's the entire daily interface.

On your screen — capture, check the lamp, recall
Terminal + Claude Code
$ capture ~/Downloads/client-call.pdf filed → Inbox/ ✓ $ boot brain OK · sync OK · recall OK READY OK # then just ask, in plain English: you › what did I say about pricing on the client call? claude › You said: "anchor at $2k, never discount the first call." — client-call.pdf

CAPTURE

  • capture <file-or-url> — files a doc, web page, PDF, or YouTube link into your brain
  • …or just drop a note in your Obsidian Inbox/

RECALL

  • Ask Claude Code in plain English: "What did I save about ___?"
  • Your own words come back, cited to the source

Check the lamp anytime with boot — four OK lines + "READY OK" means the brain is live.

It grows itself

Week one onward: /level-up

weekly · ~10 minutes · opt-in

Run it once a week. Each run ships one new workflow and improves one existing one — the system accrues capability one piece at a time, never an org chart drawn up front.

On your screen — one workflow shipped, one improved
Claude Code — /level-up
/level-up Weakest C this week: Cadence Biggest time sink: sorting podcast clips by hand → Automate. Shipped: departments/content/workflows/clip-sort/ Manager retro: tightened CONTEXT.md step 2 Logged → decisions/log.md ✓

The ritual

  • 30-sec health check on the "Four Cs" (Context · Connections · Capabilities · Cadence)
  • Find the one thing that cost you the most time this week
  • Eliminate / Automate / Delegate it → ship it into the right departments/ folder
  • That department's manager does a retro and makes one improvement

You

  • Name the week's biggest time sink
  • Set the autonomy level (draft-for-approval, or just-do-and-tell)
  • Optionally put proven workflows on a schedule
Boring is beautiful. Nothing runs unasked until a human has watched it work manually — that guardrail is on purpose.
